Installation

Last updated: August 22, 2025 3 min read

Get started with AngularUI Pro by installing the package via npm and setting up your first component.

Prerequisites

Before installing AngularUI Pro, make sure you have the following:

  • Angular 15.0.0 or higher
  • Node.js 16.0.0 or higher
  • npm 7.0.0 or higher

Package Installation

Install AngularUI Pro using npm or yarn:

Terminal 
npm install @uttam002/angular-ui-pro
Terminal 
yarn add @uttam002/angular-ui-pro
Terminal 
pnpm add @uttam002/angular-ui-pro

Module Import

Import the AngularUI Pro module in your app module:

app.module.ts 
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { AngularUiProModule } from '@uttam002/angular-ui-pro';

import { AppComponent } from './app.component';

@NgModule({
  declarations: [
    AppComponent
  ],
  imports: [
    BrowserModule,
    AngularUiProModule // Add this line
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

Styles Import

Include the CSS styles in your angular.json or styles.css:

angular.json 
"styles": [
  "src/styles.css",
  "node_modules/@uttam002/angular-ui-pro/themes/default.css"
]
styles.css 
@import '~@uttam002/angular-ui-pro/themes/default.css';

Verify Installation

Test your installation with a simple button component:

app.component.html 
<div class="app-container">
  <h1>Welcome to AngularUI Pro!</h1>
  
  <ui-button 
    variant="primary" 
    (click)="onButtonClick()">
    Click Me!
  </ui-button>
</div>
Pro Tip: Use tree-shaking by importing only the modules you need. Check our Configuration Guide for more details.

Quick Start

Last updated: August 22, 2025 5 min read

Learn how to build your first application using AngularUI Pro components with this step-by-step guide.

Creating Your First Component

Let's create a simple login form using AngularUI Pro components:

login.component.html 
<div class="login-container">
  <ui-card class="login-card">
    <ui-card-header>
      <h2>Welcome Back</h2>
      <p>Sign in to your account</p>
    </ui-card-header>
    
    <ui-card-content>
      <form [formGroup]="loginForm" (ngSubmit)="onSubmit()">
        <ui-form-field>
          <ui-label>Email</ui-label>
          <ui-input 
            type="email" 
            formControlName="email"
            placeholder="Enter your email"
            [invalid]="emailInvalid">
          </ui-input>
          <ui-error *ngIf="emailInvalid">
            Please enter a valid email
          </ui-error>
        </ui-form-field>

        <ui-form-field>
          <ui-label>Password</ui-label>
          <ui-input 
            type="password" 
            formControlName="password"
            placeholder="Enter your password"
            [invalid]="passwordInvalid">
          </ui-input>
          <ui-error *ngIf="passwordInvalid">
            Password is required
          </ui-error>
        </ui-form-field>

        <ui-button 
          type="submit" 
          variant="primary" 
          size="large"
          [loading]="isLoading"
          [disabled]="loginForm.invalid"
          class="submit-btn">
          Sign In
        </ui-button>
      </form>
    </ui-card-content>
  </ui-card>
</div>

Component Logic

Implement the component logic with form validation:

login.component.ts 
import { Component, OnInit } from '@angular/core';
import { FormBuilder, FormGroup, Validators } from '@angular/forms';

@Component({
  selector: 'app-login',
  templateUrl: './login.component.html',
  styleUrls: ['./login.component.scss']
})
export class LoginComponent implements OnInit {
  loginForm: FormGroup;
  isLoading = false;

  constructor(private fb: FormBuilder) {}

  ngOnInit() {
    this.loginForm = this.fb.group({
      email: ['', [Validators.required, Validators.email]],
      password: ['', [Validators.required, Validators.minLength(6)]]
    });
  }

  get emailInvalid() {
    const control = this.loginForm.get('email');
    return control?.invalid && control?.touched;
  }

  get passwordInvalid() {
    const control = this.loginForm.get('password');
    return control?.invalid && control?.touched;
  }

  async onSubmit() {
    if (this.loginForm.valid) {
      this.isLoading = true;
      
      try {
        // Simulate API call
        await this.delay(2000);
        console.log('Login successful!', this.loginForm.value);
      } catch (error) {
        console.error('Login failed:', error);
      } finally {
        this.isLoading = false;
      }
    }
  }

  private delay(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}

Configuration

Last updated: August 22, 2025 4 min read

Configure AngularUI Pro to match your application's requirements with custom settings and optimizations.

Global Configuration

Set up global configuration using the forRoot method:

app.module.ts 
import { AngularUiProModule, UI_PRO_CONFIG } from '@uttam002/angular-ui-pro';

@NgModule({
  imports: [
    AngularUiProModule.forRoot({
      theme: 'material-dark',
      rippleDisabled: false,
      animationDuration: 300,
      breakpoints: {
        mobile: 768,
        tablet: 1024,
        desktop: 1200
      }
    })
  ]
})
export class AppModule { }

Configuration Options

Property Type Default Description
theme string 'material' Default theme name
rippleDisabled boolean false Globally disable ripple effects
animationDuration number 250 Default animation duration (ms)
breakpoints object See example Responsive breakpoints
